### **9. Learning Objectives**
*   This microcredential introduces the core principles, tools, and methodologies for successfully managing projects from initiation to closure. Through a hands-on, project-based approach, learners will apply fundamental concepts from both traditional and Agile methodologies in a simulated environment. The curriculum is aligned with globally recognized industry standards, providing a solid foundation for aspiring project managers and professionals seeking to enhance their organizational skills.

### **10. Content Description**
*   **Module 1: Introduction to Project Management**
    *   Core terminology, roles, and responsibilities.
    *   The project lifecycle and process groups.
*   **Module 2: Traditional Project Management (Waterfall)**
    *   Project initiation: The Project Charter.
    *   Project planning: Work Breakdown Structure (WBS), scheduling, and risk registers.
*   **Module 3: Agile & Scrum Fundamentals**
    *   The Agile Manifesto and principles.
    *   The Scrum Framework: Roles, events, and artifacts.
    *   Writing user stories and creating a product backlog.
*   **Module 4: Project Execution and Monitoring**
    *   Stakeholder communication and management.
    *   Tracking progress using project management software.
    *   Managing project constraints and change.

### **11. Mode of Instruction**
*   **Fully Online (Hybrid Asynchronous/Synchronous):** Learning materials are provided asynchronously for flexible study, complemented by mandatory synchronous online workshops for collaboration and Q&A.

### **12. Learning Outcomes**
*   Upon successful completion of this microcredential, the learner will be able to:
    *   **LO1:** Describe and compare the fundamental principles, key terminology, and lifecycles of both traditional (Waterfall) and Agile project management.
    *   **LO2:** Apply core project management tools and techniques for planning, executing, and monitoring a project, including the creation of a Work Breakdown Structure (WBS) and an Agile product backlog.
    *   **LO3:** Analyze project requirements and manage stakeholder expectations in a simulated environment.
    *   **LO4:** Evaluate common project risks and propose basic mitigation strategies.
    *   **LO5:** Collaborate effectively within a team to deliver a comprehensive project plan for a given case study.

### **15. Learning Activities**
*   The learning experience is structured around a central team-based project and includes:
    *   **Interactive e-modules & readings:** Self-paced learning of core concepts.
    *   **Case study analysis:** Applying theory to real-world scenarios.
    *   **Team-based hybrid project:** A course-long project where teams first use traditional methods (Project Charter, WBS) and then transition to Agile sprints using a product backlog. The project includes a simulated **"change request"** mid-sprint to develop adaptability.
    *   **Live online workshops:** Collaborative sessions for Q&A, peer feedback, and instructor guidance.
    *   **Final project development & presentation:** Consolidating all project artifacts and presenting findings.

### **16. Workload Theoretical**

### **18. Assessment Methods**
*   **Summative Assessment (100%):**
    *   **Final Team Project (60%):** A comprehensive project portfolio submission, including:
        *   A Project Charter and Work Breakdown Structure (WBS).
        *   An Agile Product Backlog with User Stories.
        *   A link to a configured project management tool board (e.g., Trello) showing sprint planning and progress.
        *   A recorded team presentation that includes a sprint retrospective.
    *   **Individual Reflection Paper (20%):** A critical reflection on the project process, comparing the use of traditional and Agile approaches and the team's response to the change request.
    *   **Quizzes and Participation (20%):** Based on formative quizzes on core concepts and active participation in workshops.
*   A passing grade of 50% is required to earn the microcredential.
