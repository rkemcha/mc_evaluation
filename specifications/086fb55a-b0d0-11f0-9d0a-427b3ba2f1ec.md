## 9. Learning Objectives

Upon successful completion of this microcredential, learners will be able to:
*   Navigate the Linux file system and manage files and directories using command-line tools.
*   Manage users and groups, including setting permissions and access control for files and directories.
*   Install, update, and manage software packages on a Linux system.
*   Monitor and manage running processes and system resources.
*   Configure basic network settings and troubleshoot connectivity issues.
*   Utilize essential Linux command-line tools for system querying and information gathering.
*   Develop and execute basic shell scripts for automating repetitive administrative tasks.
*   Perform basic troubleshooting of common Linux system issues.

**Reasoning Process & Justification:**
*   **Reasoning:** Derived from the `Topic: Administration of Linux systems` and `Level: Foundational`. The outcome for shell scripting directly integrates the selected trend, empowering learners with automation capabilities. These outcomes focus on core, practical skills expected of an entry-level Linux administrator, covering file management, user management, package management, process control, basic networking, troubleshooting, and **foundational automation**. The outcome on system querying was refined for clarity.
*   **Justification:** These outcomes are specific, measurable, achievable, relevant, and time-bound (SMART). They directly address the foundational skills required to effectively administer a Linux system, **including the highly valued skill of basic automation**, preparing learners for practical application and further learning.

## 10. Content Description

The microcredential is structured into seven modules, progressing from fundamental Linux concepts to practical system administration tasks and basic automation with shell scripting. Each module includes theoretical instruction, guided hands-on labs, and problem-solving challenges.

---

**Module 1: Introduction to Linux and the Command Line**
*   What is Linux? History and distributions.
*   Accessing the Linux command line (CLI).
*   Basic command syntax and help (`man`, `info`).
*   Navigating the file system (`pwd`, `ls`, `cd`).
*   Managing files and directories (`mkdir`, `rm`, `cp`, `mv`, `touch`).
*   Using text editors (Nano, Vim basics).
*   **Hands-on Lab:** Basic CLI navigation and file management.

**Module 2: Users, Groups, and Permissions**
*   Linux user and group concepts.
*   Creating, modifying, and deleting users and groups.
*   File and directory permissions (`chmod`, `chown`).
*   Special permissions (SUID, SGID, Sticky Bit - conceptual).
*   **Hands-on Lab:** User/group management and permission configuration.

**Module 3: Software Management**
*   Understanding package managers (`apt`, `yum`, `dnf`).
*   Installing, updating, and removing software packages.
*   Managing software repositories.
*   Basic dependency resolution.
*   **Hands-on Lab:** Installing and managing software.

**Module 4: System Processes and Resource Monitoring**
*   Understanding processes and their lifecycle.
*   Managing processes (`ps`, `top`, `htop`, `kill`).
*   Managing services (`systemctl`).
*   Monitoring system resources (CPU, memory, disk usage).
*   **Hands-on Lab:** Process and service management, system monitoring.

**Module 5: Basic Networking**
*   Network interface configuration (`ip`, `ifconfig`).
*   Network troubleshooting tools (`ping`, `netstat`, `ss`, `traceroute`).
*   DNS resolution.
*   Introduction to firewalls (`ufw`, `firewalld` basics).
*   **Hands-on Lab:** Configuring network interfaces and troubleshooting connectivity.

**Module 6: System Querying and Information Gathering**
*   Utilizing command-line tools for filtering and processing text (`grep`, `find`, `cut`, `awk` basics).
*   Gathering system information (`uname`, `hostname`, `df`, `free`, `ip`).
*   Combining commands with pipes and redirects.
*   **Hands-on Lab:** Advanced command-line usage for data extraction and system information.

**Module 7: Shell Scripting Fundamentals and Automation**
*   Introduction to Bash scripting.
*   Variables, input/output, and quoting.
*   Conditional statements (`if`, `else`, `elif`).
*   Looping constructs (`for`, `while`).
*   Functions in scripts.
*   Creating simple automation scripts (e.g., backup script, user creation script, system health check).
*   Scheduling tasks with Cron.
*   **Hands-on Lab:** Writing and executing basic shell scripts, automating a simple administrative task.

**Module 8: System Logging and Basic Troubleshooting**
*   Understanding system logs (Syslog, Journald).
*   Viewing and analyzing log files.
*   Common troubleshooting methodologies.
*   Diagnosing common Linux system issues.
*   **Hands-on Lab:** Analyzing logs to troubleshoot simulated system issues.

## 18. Assessment Methods

*   **Quizzes/Short Assignments:** To assess conceptual understanding of Linux commands, system components, administrative principles, **and basic scripting syntax/logic.**
*   **Practical Lab Exercises:** Hands-on tasks completed in a virtual Linux environment, demonstrating proficiency in command-line operations, user/file management, package installation, basic troubleshooting, **and the creation/execution of simple shell scripts.**
*   **Final Project:** A comprehensive practical assignment involving the setup, configuration, and basic administration of a small Linux system (e.g., a web server or file server), **including the automation of at least one administrative task using a shell script.**

**Reasoning Process & Justification:**
*   **Reasoning:** Derived from the `Level: Foundational`, `Pedagogical Approaches` (especially Project-Based Learning and Active Learning), and `Industry Relevance` (hands-on). Assessments must verify both theoretical knowledge and practical application, **now explicitly including the ability to write and use basic scripts.**
*   **Justification:** A combination of theoretical quizzes and practical lab exercises/projects ensures a holistic evaluation of learning outcomes, reflecting both knowledge acquisition and the ability to apply skills in real-world scenarios, which is critical for system administration, **and now explicitly for basic automation.**
