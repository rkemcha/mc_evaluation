## **3. Learning Objectives**
By completing this microcredential, learners will achieve the following competencies:

1. **Define Kubernetes Architecture**
   - Understand core components: Pods, Nodes, Services, Controllers, and API Server.
   - Deploy a Kubernetes cluster using Minikube/Kind.

2. **Deploy and Manage Containers**
   - Use `kubectl` commands (`create`, `delete`, `apply`) to manage Pods and Deployments.
   - Integrate Docker images into Kubernetes workloads.

3. **Configure Helm Charts**
   - Create and customize Helm charts for containerized applications.
   - Deploy Helm charts using `helm install` and `helm upgrade`.

4. **Optimize Resource Usage**
   - Set CPU/memory requests and limits for Pods.
   - Configure Horizontal Pod Autoscaler (HPA) for dynamic scaling.

5. **Troubleshoot Common Issues**
   - Diagnose Pod crashes (e.g., `CrashLoopBackOff`, `OOMKilled`).
   - Use `kubectl logs` and `kubectl describe` for debugging.

6. **Integrate Monitoring and Logging**
   - Deploy Prometheus for metrics collection and Grafana for visualization.
   - Centralize logs using the EFK stack (Elasticsearch, Fluentd, Kibana).

7. **Apply CI/CD Principles**
   - Implement GitOps workflows using ArgoCD.
   - Automate Kubernetes deployments with GitHub Actions.

## **4. Content Description**
### **Module 1: Kubernetes Basics + Istio (12 hours)**
**Objective:** Understand Kubernetes architecture and deploy a basic cluster with Istio Sidecar injection.
**Activities:**
- **Lecture (2h):** Pods, Nodes, Services, and Istio components (Sidecar, VirtualServices).
- **Hands-on Lab (6h):**
  - Install Minikube with Istio add-on.
  - Deploy a simple Pod and expose it via a Service.
  - Configure Istio Sidecar injection for a custom app.
- **Case Study (4h):**
  - *Scenario:* Deploy a web app (e.g., Nginx) with Istio traffic routing.
  - *Task:* Write YAML manifests for Pods, Services, and Istio VirtualServices.

**Assessment:**
- **Quiz (2h):** 10 MCQs on Pods, Services, and Istio concepts.
- **Lab Report:** Screenshots + code snippets for the case study.

---

### **Module 2: Advanced Deployment + Istio (12 hours)**
**Objective:** Master Istio traffic management and CI/CD pipelines.
**Activities:**
- **Lecture (2h):** Istio VirtualServices, Traffic Splitting, and Canary Deployments.
- **Hands-on Lab (6h):**
  - Deploy a stateless app (e.g., Flask) with Istio VirtualServices.
  - Configure traffic splitting (e.g., 50% to new version, 50% to old).
  - Explore ArgoCD for GitOps workflows.
- **Project (4h):**
  - *Task:* Design a Kubernetes cluster for a microservice (e.g., order service + payment service).
  - *Deliverable:* Architecture diagram + `kubectl apply` commands + Istio VirtualServices.

**Assessment:**
- **Project Submission:** Cluster design + code snippets.
- **Live Demo (2h):** Present traffic management strategies to peers.

---

### **Module 3: Helm, Monitoring, and Network Policies (4 hours)**
**Objective:** Learn Helm templating, monitoring, and security policies.
**Activities:**
- **Lecture (1h):** Helm basics (repositories, values files, hooks).
- **Hands-on Lab (3h):**
  - Create a Helm chart for a custom app (e.g., Redis).
  - Deploy using `helm install` and `helm upgrade`.
- **Case Study (2h):**
  - *Scenario:* Integrate Helm into a GitHub Actions pipeline.
  - *Task:* Write a CI/CD workflow for Kubernetes deployments.

**Assessment:**
- **Helm Challenge:** Deploy a chart with custom values and document the process.
